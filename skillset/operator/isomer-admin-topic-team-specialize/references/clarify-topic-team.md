# Clarify Topic Team

## Workflow

When this subcommand is selected, execute the following steps in order.

1. Read the specialization outputs: topic overview, copied template root, `team-specialization-guide.md`, `team-specialization-plan.md`, `Final Report`, placeholder resolutions, deferrals, and draft profile summary.
2. Identify role, workflow, assumption, policy, binding, copied-material, or blocker questions that need user review.
3. Ask the user for focused revisions or decisions about the specialized topic team.
4. Update or report revisions to the copied material, specialization plan, `Final Report`, placeholder resolutions, deferrals, or draft profile inputs as needed.
5. Report `topic_team_revision_status`, changed paths, remaining blockers, validation refs, and whether setup can proceed.

If the user's task does not map cleanly to these steps, use your native planning tool to build a step-by-step revision plan from the specialization outputs, requested changes, and guardrails, then execute the plan.

## Guardrails

Do not approve, materialize, or launch the team from this subcommand.

Do not change the source Domain Agent Team Template.

Do not hide unresolved launch blockers. Keep deferrals explicit.
