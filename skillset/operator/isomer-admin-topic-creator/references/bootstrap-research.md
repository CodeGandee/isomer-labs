# Bootstrap Research

## Workflow

When this command is selected, execute the following steps in order.

1. Require registered Topic Workspace, valid Workspace Runtime, topic overview, topic environment readiness, ready `topic.repos.main`, materialized research record labels, and selected Topic Actor bindings.
2. Invoke or validate `isomer-rsch-workspace-mgr-v2` for base topic readiness and selected Topic Actor readiness.
3. Validate selected v2 skill `placeholder-bindings.md` files, topic-level placeholder binding index or readiness report, actor recording metadata, and accepted-artifact command guidance.
4. Treat missing actor or formal team layer checks as topology-specific blockers, not as proof that the whole topic is unusable.
5. Report bootstrap status, placeholder binding entrypoints, storage recording command shapes, blockers, and next command.

If the user's task does not map cleanly to these steps, name whether the blocker is base topic readiness, Topic Actor readiness, v2 binding readiness, or optional formal team readiness.

## Guardrails

Do not write accepted research artifacts only to local scratch, `topic.repos.main`, Topic Actor Workspace files, or Agent Workspace files without recording or linking them through Topic Workspace research records.
