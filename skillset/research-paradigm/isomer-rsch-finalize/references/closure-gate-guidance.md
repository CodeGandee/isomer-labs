# Closure Gate Guidance

Use this reference when deciding whether the final outcome should be stop, park, publish, archive, continue, reset, new loop, or Operator Agent approval.

## Closure Choices

| Choice | Use when | Required record |
| --- | --- | --- |
| stop | The current objective should end because the result is complete, blocked beyond current scope, or no longer valuable. | Final Decision Record and reopen condition |
| park | The current objective should pause with a clean resume path. | Resume packet and pause rationale |
| publish | The deliverable is ready enough for release, manuscript, report, or external handoff. | Package inventory and completion Gate |
| archive | The work should be preserved without further action now. | Final report and inventory |
| continue | A bounded next stage remains justified. | Decision Record routing back to the proper skill |
| reset | The current line is invalid and a new frame is needed. | Decision Record and scout or idea handoff |
| new loop | The current loop produced a new incumbent or route for a separate cycle. | Handoff packet and new Research Task recommendation |

## Operator Agent Gate

Run Gate Policy preflight for true completion, publication-facing output, finality, privacy, cost, scope, safety, data export, external upload, credential use, or irreversible archive decisions, and open an Operator Agent Gate when the selected policy requires human judgment. Do not use a Gate for routine missing evidence; route that back through decision, analysis, write, review, or baseline.

## Paper or Report Closure

Do not finalize a paper-like line while main-text coverage, evidence mapping, review, proofing, language, or submission package checks still block the deliverable. A draft checkpoint is not a final package, and a review package is not a completed rebuttal.
