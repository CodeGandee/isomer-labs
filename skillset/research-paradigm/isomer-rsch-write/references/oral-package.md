# Oral Package

Use this reference when a dense but evidence-bearing draft needs to become a more memorable, reviewer-ready paper package.

## Core Idea

An oral-quality paper is not only stronger prose. It stages evidence so a reviewer can understand the idea quickly, inspect the main claim, remember one central lesson, and see that likely objections were anticipated.

## Persuasion Chain

A strong package usually presents: context and why the problem matters; deficiency in current methods; the core idea in plain language; an early intuition or mechanism anchor; the main empirical result; targeted analysis explaining why the result is credible; and appendix material that supports deeper verification.

## Evidence Budget

Spend main-text space where reviewer friction is highest. Preserve a visible main-result surface, at least one interpretive analysis or tradeoff block when the claim needs explanation, and one practical-value or objection-handling block when the paper invites that concern. Use appendix for full tables, protocols, broader sweeps, proofs, extended examples, extra failure cases, and reproducibility details.

## Display Program

Assign each display one job: intuition or mechanism, main result, interpretive analysis, practical value, robustness, failure boundary, or case study. Displays carry exact values, full comparisons, traces, and concrete examples. Surrounding prose states the question, dominant pattern, interpretation, and reviewer-facing implication.

## Main Text Versus Appendix

Move exhaustive evidence to appendix, but keep claim-critical support inspectable in the main text. Generic appendix bridges are weak. Name the overflow job: protocol detail, full benchmark table, tuned baseline, transfer sweep, extended literature, qualitative examples, failure cases, or reproducibility material.

## Oral Delta Map

Before line editing, write what the current draft lacks: reader onboarding, early intuition, display roles, separate experiments and analysis, memorable case study, appendix support, claim calibration, citation support, or artifact consistency. When two versions exist, preserve changes that improve persuasion, not changes that merely add length.

## Failure Modes

Flag overloaded abstracts, introductions that jump to wins, method sections that only define equations, results that read like spreadsheets, analysis that repeats visible numbers, figures used as storage, pages with too many local claims, appendix dumps, weak signposting, inconsistent artifact availability, and claim language that outruns evidence.
