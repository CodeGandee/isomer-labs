# Evaluation Contract Template

Use this template when scout needs a durable evaluation contract that baseline, idea, experiment, analysis, and writing can reuse without re-deriving core assumptions.

```md
# Evaluation Contract

- Research Task:
- task:
- dataset:
- dataset version or source:
- split contract:
- official or expected evaluation path:
- primary metric:
- metric direction:
- secondary metrics:
- fair-comparison rule:
- useful-improvement threshold:
- minimum evidence needed for a claim:

## Evidence
- source Artifacts:
- benchmark documents:
- paper or repository references:
- local Findings:

## Known Ambiguities
- ambiguity:
- why it matters:
- blocked Workflow Stage:
- Gate or user choice needed:

## Decision Impact
- baseline implication:
- idea implication:
- experiment implication:
- blocker:
```

The contract should be concise, concrete, and honest about missing pieces. If multiple incompatible evaluation contracts remain plausible, scout should recommend a Gate or decision instead of silently picking one.
