# Fusion Playbook

Use fusion only when at least two lines have real strengths, the strengths are complementary, and one line alone is no longer improving fast enough.

## Pre-Fusion Record

```md
# Fusion Record

## Source Line A

- strongest_mechanism:
- strongest_evidence:
- main_weakness:
- what_must_survive_the_fusion:

## Source Line B

- strongest_mechanism:
- strongest_evidence:
- main_weakness:
- what_must_survive_the_fusion:

## Fusion Hypothesis

- what_exactly_is_being_fused:
- bottleneck_addressed:
- why_strengths_are_complementary:
- keep_unchanged_for_comparability:
- evidence_that_would_prove_fusion_worthwhile:
- bounded_first_validation_step:
```

## Do Not Fuse

- two lines with the same mechanism under different names
- two weak lines with no clear strengths
- lines that only coexist because multiple Research Inquiry Relationships exist

## Exit Rule

If the fusion hypothesis is underspecified, return to brief shaping instead of pretending fusion is ready.
