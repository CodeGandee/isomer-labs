# High-Value Idea Sourcing

Use this reference when ideation is in danger of starting from a clever technique rather than an important contradiction. Placeholder definitions live in `../migrate/placeholders.md`.

## Guidance

When performing this step, execute these substeps in order.

1. **Name the contradiction**. Identify the unresolved anomaly, failure region, evaluation distortion, theoretical rupture, or neglected problem slice that makes the route worth considering.
2. **Decompose novelty**. Classify whether the plausible contribution is a new problem, view, mechanism, method, setting, evaluation lens, or boundary condition.
3. **Use multiple source lenses**. Check gap-driven, anomaly-driven, bottleneck-driven, assumption-reversal, cross-domain transfer, and writing-to-think routes before committing to one family.
4. **Ask mechanism questions**. For each serious route, state why it should work, where it should fail, and which mechanism changes.
5. **Score failure value**. Keep candidates whose negative result would still teach something about the bottleneck, mechanism, or evaluation.
6. **Run the local-optimum check**. Challenge whether the apparent best route is only familiar, cheap, or already half-implemented.

## Preferences

- Prefer contradiction-first sourcing over method-first module shopping (if no contradiction is clear, otherwise stay in problem definition).
- Prefer ideas with interpretable failure value (if a route can only produce noise on failure, otherwise downgrade it).
- Prefer non-local or assumption-reversal alternatives when the current mainline has hidden untested assumptions (if the mainline is already decisive, otherwise record why widening is unnecessary).

## Constraints

- A serious candidate must name its targeted contradiction or bottleneck.
- A serious candidate must state a dominant novelty source or research-value source.
- A serious candidate must expose its core hypothesis, mechanism sketch, and strongest falsification path.
- The idea pass must not treat the easiest continuation of the incumbent as best by default.

## Quality Gates

- Contradiction quality: the target problem is important to the real objective, not merely convenient to implement.
- Novelty quality: the dominant novelty or value source is explicit and honest.
- Mechanism quality: the route says what changes and what condition would make it fail.
- Lock-in resistance: at least one outside-family or assumption-reversal alternative remains visible until selection.
