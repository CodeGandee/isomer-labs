# Campaign Design

Use this reference to design the smallest analysis campaign that can change a parent claim or route. Placeholder definitions live in `../migrate/placeholders.md`.

## Workflow

When this reference is used, execute the following steps in order.

1. **Name the parent object and evidence question**.
2. **Select slice classes: ablation, robustness, failure, limitation, qualitative, or writing-facing**.
3. **Bound resources before execution**.
4. **Reject slices that cannot change interpretation**.
5. **End with the slice set and stop condition**.

If the user's task does not map cleanly to these steps, use your native planning tool to build a step-by-step plan from this reference, the parent skill, and the available evidence, then execute the plan.
