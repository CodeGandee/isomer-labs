# Analysis Plan Roles

Use this reference when choosing reviewer-facing analysis jobs for a mature outline.

## Role Catalog

- component ablation: tests whether the claimed component causes the gain.
- robustness or sensitivity: tests whether the claim survives settings, datasets, scales, or perturbations.
- stronger-baseline comparison: tests whether the gain survives a fairer or retuned comparator.
- subgroup or case breakdown: shows where the method works or fails.
- failure taxonomy: names failure modes and scope limits.
- mechanism or attribution check: explains why the pattern appears.
- cost, budget, or efficiency tradeoff: measures practical constraints and tradeoffs.
- limitation or residual headroom analysis: states where the method stops and what remains unprobed.

## Selection Rule

Choose analysis jobs because they answer likely reviewer questions and protect scoped claims, not because a fixed checklist demands them.

## Count Guidance

Full empirical mature outlines usually need four to eight planned analysis jobs. If fewer are feasible, mark the outline as seed or narrow, or include an analysis-budget waiver that names the consequence for claim scope.

## Display Planning

Each analysis job should name its target display, manuscript placement, linked claims, reviewer question, and failure interpretation. Do not leave all analysis as appendix-only if the central claim depends on visible interpretation.

## Failure Interpretation

Each analysis job should say how the claim changes if the result fails: downgrade, limitation, appendix-only support, alternative explanation, or route-back to baseline or experiment.
