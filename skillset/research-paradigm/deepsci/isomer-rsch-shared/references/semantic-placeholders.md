# Production DeepSci Semantic Placeholders

This registry defines bind-able semantic research artifacts for production DeepSci skills. It explains what each artifact means, what content it must carry, which skills usually produce it, and which skills usually consume it.

This page intentionally does not track whether an artifact is currently bound to a concrete storage layer. Concrete binding details such as placeholder tokens, record kinds, semantic labels, format profiles, and create or update command shapes belong in the producing skill's `placeholder-bindings.md` file.

| ID | Bind-able artifact | Meaning | Required semantic content | Typical producers | Typical consumers |
| --- | --- | --- | --- | --- | --- |
| `workspace-bootstrap` | Workspace bootstrap contract | The post-specialization production DeepSci research workspace readiness contract. | Topic refs, profile bundle readiness, runtime readiness, semantic surface plan, placeholder binding registry, Agent Workspace access posture, validation status, and blockers. | `isomer-rsch-workspace-mgr` | All production DeepSci research skills, Topic Service Master, Project Operator Session, and Operator Agent |
| `latest-context-snapshot` | Latest context snapshot | A storage-neutral freshness verdict for accepted durable record work. | Selected Research Topic refs, Research Topic Config source, Topic Workspace path or absence, Workspace Runtime inspection status, effective Topic Actor or Agent context, checked record refs, freshness verdict, prompt-versus-durable-context relation, conflict route, and blockers. | Any active production DeepSci skill with durable record bindings, usually in its first context brief, contract, route decision, paper state, or blocker record | Any production DeepSci skill before accepting durable record writes, record refreshes, durable route decisions, or durable claim/context/evidence/result decisions |
| `research-frame` | Research frame | The scoped research problem and success boundary. | Problem statement, objective, constraints, metric or evaluation need, known unknowns, and next likely stage. | `isomer-rsch-scout`, `isomer-rsch-decision` | `isomer-rsch-baseline`, `isomer-rsch-idea`, `isomer-rsch-finalize` |
| `comparator-contract` | Comparator contract | The comparator and measurement basis that make later progress meaningful. | Comparator identity or waiver reason, metric definition, comparability assumptions, accepted deviations, and blockers. | `isomer-rsch-baseline`, `isomer-rsch-decision` | `isomer-rsch-idea`, `isomer-rsch-experiment`, `isomer-rsch-analysis` |
| `selected-hypothesis` | Selected hypothesis | One falsifiable route chosen from the current research situation. | Hypothesis, expected effect, mechanism sketch, novelty boundary, risk, falsification condition, and next experiment. | `isomer-rsch-idea`, `isomer-rsch-optimize`, `isomer-rsch-decision` | `isomer-rsch-experiment`, `isomer-rsch-optimize`, `isomer-rsch-analysis` |
| `optimization-frontier` | Optimization frontier | The candidate set and ranking logic for algorithm-first search. | Candidate briefs, ranking criteria, current best line, rejected or parked ideas, fusion/debug notes, and next selected route. | `isomer-rsch-optimize` | `isomer-rsch-idea`, `isomer-rsch-experiment`, `isomer-rsch-decision` |
| `experiment-contract` | Experiment contract | The bounded plan that makes one test interpretable. | Selected hypothesis, comparator basis, metric, input conditions, command or procedure summary, expected outputs, and stop condition. | `isomer-rsch-experiment`, `isomer-rsch-science` | `isomer-rsch-experiment`, `isomer-rsch-analysis`, `isomer-rsch-decision` |
| `experiment-result` | Experiment result | The measured outcome of one bounded test. | What was run, observed metrics, important outputs, failures or anomalies, comparator relation, and interpretation. | `isomer-rsch-experiment`, `isomer-rsch-science` | `isomer-rsch-analysis`, `isomer-rsch-decision`, `isomer-rsch-finalize` |
| `analysis-finding` | Analysis finding | A focused interpretation that changes or tests a result or claim. | Parent result, question answered, slice or check performed, finding, caveat, claim impact, and next route. | `isomer-rsch-analysis`, `isomer-rsch-science` | `isomer-rsch-decision`, `isomer-rsch-finalize`, `isomer-rsch-idea` |
| `science-validity-note` | Science validity note | A computation, package, data, simulation, or validation note that affects trust. | Scientific task, inputs, assumptions, tool or method summary, checks performed, validity verdict, and limits. | `isomer-rsch-science` | `isomer-rsch-experiment`, `isomer-rsch-analysis`, `isomer-rsch-decision` |
| `route-decision` | Route decision | A justified research routing choice. | Decision question, considered options, chosen route, evidence basis, rejected alternatives, blocker if any, and next action. | `isomer-rsch-decision`, any production DeepSci skill when the route is obvious | Any production DeepSci skill |
| `final-summary` | Final summary | The responsible end-state summary for pause, closure, handoff, or publication-facing work. | Final position, supported claims, limitations, unresolved risks, useful outputs, recommended next action, and stop or handoff reason. | `isomer-rsch-finalize` | Maintainers, downstream agents, future write/review skills |
