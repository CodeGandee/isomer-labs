# Optimize Checklist

Use this checklist for a non-trivial optimize pass.

```md
# Optimize Checklist

- [ ] Read the durable frontier summary or equivalent View Manifest.
- [ ] Select the primary optimize submode: brief, rank, seed, loop, fusion, debug, or stop.
- [ ] Confirm whether the current pass is explore, exploit, fusion, debug, or stop.
- [ ] Review recent Findings before generating new candidates.
- [ ] Check whether the method-brief slate covers more than one mechanism family.
- [ ] Candidate briefs updated or confirmed.
- [ ] Candidate ranking updated.
- [ ] Promote only the strongest brief or briefs into durable Research Inquiry Relationships when justified.
- [ ] Current implementation-attempt pool recorded.
- [ ] Smoke or pilot queue defined when execution is needed.
- [ ] Full-evaluation queue defined when justified.
- [ ] Recent failures classified and either debugged, archived, or routed.
- [ ] Stagnation check performed.
- [ ] Family-shift trigger checked.
- [ ] Fusion eligibility checked.
- [ ] Next concrete action written.
```

## Closeout Rule

An optimize pass is not done until one durable next action, stop condition, or blocker is recorded.
