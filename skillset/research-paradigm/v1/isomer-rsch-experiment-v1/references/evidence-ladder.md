# Evidence Ladder

Use this reference when deciding whether an experiment package is only minimally acceptable or strong enough to carry a Research Claim.

## Core Ladder

- `minimum`: executable result, comparable setup, and enough evidence to show the direction is not obviously broken.
- `solid`: credible main comparison, strong and fair comparator, stable enough result to support the main claim, and significance or uncertainty checks when superiority is claimed.
- `maximum`: the main claim is already credible, and additional analysis broadens confidence, interpretation, or scope.

## Auxiliary vs Main

- `auxiliary/dev`: parameter effects, diagnostics, mechanism checks, and setup clarification.
- `main/test`: claim-carrying comparison and the evidence likely to anchor a report or paper.

Do not confuse auxiliary evidence with the main comparison.

## Default Policy

Before spending heavily on maximum polish, first move the line from minimum to solid. If a result is still below solid, the next best action is usually strengthening comparability, repairing a confounder, adding significance or uncertainty testing, or running the most claim-critical follow-up comparison.
