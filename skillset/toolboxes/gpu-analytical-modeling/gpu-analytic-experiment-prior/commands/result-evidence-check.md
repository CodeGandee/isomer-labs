# Result Evidence Check

Before experiment results are accepted, classify each result as real hardware timing, profiler counter evidence, microbenchmark, hardware query, simulator trace, emulator output, synthetic data, derivation-only evidence, or analytical proxy.

Report predicted component or path, intended stressor, observed dominant component or path, predicted latency or runtime, measured latency or runtime, raw observation summary, evidence mapping rationale, match or miss, interpretation, and explicit gap when evidence is unavailable.

Do not substitute emulator, simulator, synthetic, analytical-proxy, or derivation-only evidence for measured hardware accuracy. If NCU or another profiler cannot run, record the concrete blocker and route to repair, alternate measurement, limitation, decision, or blocker.
