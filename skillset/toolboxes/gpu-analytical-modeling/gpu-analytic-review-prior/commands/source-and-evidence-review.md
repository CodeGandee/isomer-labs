# Source and Evidence Review

At review start, list central claims and map each to its source family: local topic record, kernel source, hardware documentation, profiler counter, measurement, simulator structure, literature, or explicit assumption.

Check that each source can justify the claim. Kernel code can justify implementation behavior; hardware docs can justify nominal architecture facts; counters can support observed activity with mapping rationale; measurements can support behavior within recorded scope; simulators can suggest structure, not target-hardware truth.

Preserve source limits, missing metadata, and evidence gaps rather than rewriting them as confidence.
