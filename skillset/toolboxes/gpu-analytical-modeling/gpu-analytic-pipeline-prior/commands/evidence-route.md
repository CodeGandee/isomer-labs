# Evidence Route

Before starting a pipeline pass, identify the weakest missing support for the current GPU model: source provenance, hardware-model contract, fair baseline, evaluation contract, profiler-counter evidence, real hardware timing, component/path proof, mismatch decision, claim gate, or closure evidence.

Choose the next DeepSci stage that repairs that gap. Do not run a broad pipeline pass when a narrower stage can resolve the missing source, measurement, counter, model-shape, or claim-strength issue.

Carry forward the evidence class so proxy evidence does not become measured-hardware support later in the pipeline.
