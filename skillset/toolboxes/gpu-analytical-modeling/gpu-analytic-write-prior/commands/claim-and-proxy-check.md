# Claim and Proxy Check

Before accepting GPU analytical-modeling prose, list central claims and classify each as supported, partially supported, unsupported, or deferred. Name the evidence source for each claim: derivation, local source, real timing, profiler counter, microbenchmark, simulator, emulator, synthetic data, analytical proxy, or limitation record.

Do not describe emulator-only, simulator-only, synthetic, analytical-proxy, or derivation-only validation as measured hardware accuracy. Use wording that preserves measured scope, proxy support, explicit limitation, or a route back to experiment.

For formulas, choose concise symbols, define notation near first use, keep units explicit, and separate code identifiers from mathematical notation. Split long max expressions, tables, or metric names when they would overflow the intended document shape.
