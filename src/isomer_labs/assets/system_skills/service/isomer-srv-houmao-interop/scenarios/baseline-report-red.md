# Baseline Pressure Report: Houmao Loop Customization for DeepScientist Stage Skills

## One-Paragraph Summary of the Houmao Agent Loop

Houmao's loop is a gateway-driven request queue plus TUI-tracking lifecycle kernel. The live gateway (`extern/orphan/houmao/src/houmao/agents/realm_controller/gateway_service.py`) persists incoming work in a SQLite-backed `gateway_requests` queue, promotes accepted rows into active execution, and dispatches prompts to one managed agent at a time through a backend adapter (TUI, local headless, server headless, or CAO REST). A separate mail-notifier thread and reminder scheduler wake agents when open mailbox messages or timed reminders arrive, while a `SingleSessionTrackingRuntime` observes the managed terminal's public state. Agents do not run an always-awake inner loop; each turn is a bounded prompt-triggered pass that ends so the gateway can deliver the next notifier, control, or handoff prompt.

## Mapping DeepScientist Stage Skills to Houmao Customization Concepts

DeepScientist's native runner treats each `skill_id` (scout, baseline, idea, experiment, write, finalize) as a prompt package. The runner loads `extern/orphan/DeepScientist/src/skills/<skill_id>/SKILL.md` and its `references/`, injects them into one PI-agent prompt via `PromptBuilder`, and executes the turn through a CLI runner (Codex, Claude, Kimi, or OpenCode). To move this under Houmao, each DeepScientist stage skill is projected onto a dedicated Houmao managed agent. The agent's behavior is declared by a **role**, made launchable by a **recipe** (also called a preset), optionally registered as a **specialist** and **project profile**, and invoked at runtime through a **launch profile**. The complete multi-stage research workflow is authored as a Houmao **loop package** (`houmao-agent-loop-pro` or `houmao-agent-loop-lite`) that generates `intention/` and `execplan/` material.

### What each Houmao concept does

- **Role** (`extern/orphan/houmao/.houmao/agents/roles/<role>/system-prompt.md`): prompt-only behavior. This is where the DeepScientist `SKILL.md` body is adapted into a system prompt.
- **Recipe / preset** (`extern/orphan/houmao/.houmao/agents/presets/<name>.yaml`): binds a role to a tool lane, setup, auth bundle, and skills list. Managed by `houmao-mgr internals native-agent recipes ...` (`presets` is a compatibility alias).
- **Specialist**: a project-level catalog entry (stored in `extern/orphan/houmao/.houmao/catalog.sqlite`, authored via `houmao-mgr project specialist ...`) that points to a recipe and adds tool/credential/model defaults.
- **Project profile** (also in `catalog.sqlite`, authored via `houmao-mgr project profile ...`): a specialist-backed ready-to-launch record.
- **Launch profile** (`extern/orphan/houmao/.houmao/agents/launch-profiles/<name>.yaml`): durable launch defaults that reference a recipe or specialist; managed by `houmao-mgr project launch-profiles ...`.
- **Loop package**: a `houmao-agent-loop-pro` or `houmao-agent-loop-lite` package with `intention/` source and generated `execplan/` contracts, skills, agent bindings, and harness.

## Concrete Files an Isomer Operator Would Edit to Add a New Stage Skill

To add a new DeepScientist-style stage skill (for example, `analysis-campaign`) and run it under Houmao:

1. **DeepScientist skill source**
   - Create `extern/orphan/DeepScientist/src/skills/<new-stage>/SKILL.md` with frontmatter `name`, `description`, and `skill_role: stage`.
   - Add supporting playbooks under `extern/orphan/DeepScientist/src/skills/<new-stage>/references/`.
   - If a UI manifest is needed, add `extern/orphan/DeepScientist/src/skills/<new-stage>/agents/openai.yaml`.

2. **Houmao role**
   - Add `extern/orphan/houmao/.houmao/agents/roles/<new-role>/system-prompt.md`, derived from the `SKILL.md` body.

3. **Houmao recipe (preset)**
   - Add `extern/orphan/houmao/.houmao/agents/presets/<new-recipe>.yaml` selecting `role`, `tool`, `setup`, `skills`, `auth`, and optional `launch`/`mailbox`/`extra` fields.

4. **Houmao specialist and project profile** (project layer)
   - Author via `houmao-mgr project specialist create --name <specialist> --tool <tool> --credential <cred> ...`.
   - Author via `houmao-mgr project profile create --name <profile> --specialist <specialist> ...`.
   - These are persisted in `extern/orphan/houmao/.houmao/catalog.sqlite`, not as standalone files.

5. **Houmao launch profile**
   - Add `extern/orphan/houmao/.houmao/agents/launch-profiles/<new-profile>.yaml` referencing the new recipe/specialist, or use `houmao-mgr project launch-profiles add --name <profile> --recipe <recipe> ...`.

6. **Loop package** (if using a generated team workflow)
   - Edit `<loop-dir>/intention/loop-overview.md`, `<loop-dir>/intention/participants.md`, and `<loop-dir>/intention/workflow.md` to include the new stage and owner role.
   - Regenerate `<loop-dir>/execplan/` via `houmao-agent-loop-pro execplan-fast-forward <loop-dir>` or the lite equivalent.
   - Review/update generated files such as `<loop-dir>/execplan/agents/bindings.toml`, `<loop-dir>/execplan/specs/participants/participants.toml`, and `<loop-dir>/execplan/skills/`.

7. **Adapter/runtime linkage** (Isomer side)
   - Update `runtime/adapters/houmao/<agent-team-instance-id>/launch-material-manifest.json` and adapter manifests if the new stage changes agent bindings or launch material.
