# Start Research by Agent Team

## Workflow

1. Require an explicit Topic Team Specialization request or context that names a Domain Agent Team Template or equivalent formal-team target.
2. Collect the Research Topic, Project context, formal-team artifact, and desired guided versus fast-forward posture.
3. Recommend `$isomer-op-entrypoint use topic-team to fast-forward specialization for the selected Domain Agent Team Template` or a guided `topic-team` task.
4. Explain that formal Agent Team topology does not select DeepSci or Kaoju; offer the matching extension welcome only when a research paradigm also needs selection.
5. State that the execution owner preserves profile approval, workspace, environment, validation, materialization, launch, and Houmao boundaries, while this welcome routine changes nothing.

If the task does not map cleanly to these steps, use the native planning tool to check for a named formal-team target; if none exists, recommend manual research setup or ask the user to select a Domain Agent Team Template.

## Mutation Boundary

This routine does not specialize, approve, materialize, instantiate, or launch a team, create Agent Instances, run an Execution Adapter, or mutate Workspace Runtime. Generic preparation, readiness gaps, missing summaries, and missing Agent Workspaces do not establish formal Agent Team intent.
