# Start DeepSci Research

## Workflow

1. Interpret the goal as hypothesis-driven production research, independent of manual or formal Agent Team topology.
2. Recommend `$isomer-ext-deepsci-welcome` when the user needs typical use cases, readiness guidance, or command learning.
3. When the task is concrete, match it to a public DeepSci pass and recommend `$isomer-ext-deepsci-entrypoint use <pass> to <task>` with supplied context.
4. If Project, Topic Workspace, Topic Actor or Agent workspace, environment, extension, or DeepSci bootstrap readiness is unknown, explain the missing prerequisite and recommend the public core or DeepSci entrypoint that owns recovery.
5. State the pass's expected action and mutation posture without executing it.

If the task does not map cleanly to these steps, use the native planning tool to compare hypothesis, empirical, paper, revision, rebuttal, polish, and submission patterns, then recommend the extension welcome or one concrete pass.

## Representative Passes

Use `hypothesis-pass` to drive a selected hypothesis through experimental evidence. Use `empirical-pass` for a broader empirical loop. Use `paper-pass`, `revision-pass`, `rebuttal-pass`, `polish-pass`, or `submission-pass` for the corresponding evidence-backed manuscript stage.

This routine does not install DeepSci, bootstrap its workspace, execute a pass, write durable research records, or approve later Gates.
