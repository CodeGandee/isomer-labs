# Start DeepSci Research

## Workflow

1. Interpret this usage path as optional hypothesis-driven production research, not as a choice of manual or formal Agent Team execution topology.
2. Match the goal to a DeepSci pipeline pass or research capability using **DeepSci Capability Map**.
3. Treat package-catalog presence, Project declaration, and host usability as different evidence states; use `show-extensions` when state is unclear.
4. Recommend `Use $isomer-op-entrypoint` with the concrete DeepSci goal unless accepted extension and workspace readiness already makes a direct `$isomer-deepsci-pipeline` invocation safe.
5. Route missing extension availability or compatibility to `isomer-op-system-skill-mgr`, and route missing Project, Topic Workspace, Topic Actor or Agent workspace, or DeepSci bootstrap readiness through the entrypoint to the applicable owner.
6. State the mutation boundary and next action without running research, installation, registration, bootstrap, or workspace mutation from the welcome surface.

If the user's task does not map cleanly to these steps, use your native planning tool to compare the DeepSci pipeline capabilities, extension state, workspace readiness, execution topology, and active owner routes, then recommend one safe next invocation or ask for the missing research goal.

## DeepSci Capability Map

| Goal | Public Entry Surface |
| --- | --- |
| Develop and test a hypothesis through scouting, baseline work, ideation, optimization, experiments, analysis, and decisions. | `$isomer-deepsci-pipeline hypothesis-pass` or `empirical-pass`, selected by the prepared task. |
| Draft or revise a paper from accepted evidence. | `$isomer-deepsci-pipeline paper-pass` or `revision-pass`. |
| Respond to review, polish a manuscript, or prepare submission. | `$isomer-deepsci-pipeline rebuttal-pass`, `polish-pass`, or `submission-pass`. |
| Inspect the available named passes before choosing. | `$isomer-deepsci-pipeline list-passes`. |

The pipeline may route to specialized DeepSci skills for scouting, baselines, ideation, optimization, experiments, analysis, decisions, finalization, scientific checks, writing, review, rebuttal, plotting, figure polish, data planning, presentations, or prose polishing. Do not duplicate that detailed stage index in the welcome response unless the user asks for it.

## Readiness and Safe Invocation

If the extension and workspace are ready, recommend the matching direct pipeline invocation. If readiness is unknown, recommend:

`Use $isomer-op-entrypoint with the concrete DeepSci research goal.`

The entrypoint preserves explicit extension intent, checks Project declaration and readiness, delegates missing extension state to `isomer-op-system-skill-mgr`, delegates missing DeepSci bootstrap to `isomer-deepsci-workspace-mgr`, and selects the matching pass or owner.

DeepSci can operate with manual Topic Actors or an established formal Agent Team. Do not infer Topic Team Specialization from the research paradigm, generic preparation, missing summaries, or missing Agent Workspaces.

## Mutation Boundary

This welcome path does not install or register DeepSci, bootstrap its workspace, create or mutate a Research Topic, specialize or launch an Agent Team, execute a pipeline pass, or write research records. It only explains the capability and recommends the next owner route.
