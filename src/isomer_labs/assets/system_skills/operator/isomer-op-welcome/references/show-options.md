# Show Core Options

## Workflow

1. Present the high-value use cases in **Typical Use Cases** before any exhaustive command inventory.
2. Explain that manual versus formal Agent Team selects execution topology, while DeepSci versus Kaoju selects a research paradigm.
3. Label the phrase examples as representative routing cues, not required parser keywords.
4. If the user selects a pattern, carry their context into the exact public invocation and explain its mutation posture without executing it.
5. Offer `show-command-map` only when the user wants every current core entrypoint command.

If the user's task does not map cleanly to these steps, use the native planning tool to compare the patterns, identify the smallest missing choice, and recommend one public welcome or execution invocation without mutation.

## Choice Boundary

Manual versus Agent Team answers who or what topology conducts the work. DeepSci versus Kaoju answers which research paradigm governs the work. A user may combine one topology with one paradigm when readiness exists, and choosing an extension never implies Topic Team Specialization.
