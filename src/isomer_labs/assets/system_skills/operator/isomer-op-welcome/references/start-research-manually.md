# Start Research Manually

## Workflow

1. Interpret the goal as human-orchestrated research through Topic Actors, not formal Topic Team Specialization.
2. Collect the Research Topic or intent, Project context, Topic Actor preferences, and desired guided versus fast-forward posture.
3. Recommend `$isomer-op-entrypoint use topic-create to fast-forward the requested setup` when automatic setup is explicitly requested, or a guided `topic-create` task otherwise.
4. Explain that DeepSci or Kaoju is a separate research-paradigm choice and does not follow automatically from manual topology.
5. State that the execution owner may initialize Project readiness and prepare Topic, Workspace, environment, and Topic Actor state, while this welcome routine changes nothing.

If the task does not map cleanly to these steps, use the native planning tool to decide whether the user wants manual Topic Actor research or has supplied evidence of a formal Agent Team target, then recommend the matching public command or ask for that distinction.

## Mutation Boundary

This routine does not initialize a Project, create a Research Topic, mutate a Topic Workspace, create Topic Actor Workspaces, install packages, or execute research. The exact example does not authorize those actions until the user invokes the execution entrypoint.
