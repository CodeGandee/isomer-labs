# Recommend the Next DeepSci Step

## Workflow

1. Identify the current research stage, accepted evidence, desired output, and known Project or workspace readiness.
2. Use only read-only Project, extension, or durable-record inspection that materially affects the recommendation, and announce it first.
3. Match the result to one pattern in [show-options.md](show-options.md).
4. Recommend one public DeepSci pass, one task-only DeepSci request, or one public prerequisite-recovery route.
5. State the expected action, mutation posture, blocker or Gate, and exact resume point without executing the route.

If the task does not map cleanly to these steps, use the native planning tool to separate missing goal information from missing readiness evidence, then ask for the missing decision or recommend the narrowest read-only check.

Do not fabricate readiness. Package catalog, Project declaration, public-name observation, and verified installation are different evidence levels. Research readiness also requires the relevant Topic Workspace, actor or agent workspace, environment, and DeepSci bootstrap state.
