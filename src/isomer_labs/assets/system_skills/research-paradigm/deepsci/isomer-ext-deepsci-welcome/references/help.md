# DeepSci Welcome Help

## Workflow

1. Explain that DeepSci welcome teaches production-research patterns and remains read-only, while the DeepSci entrypoint executes passes and focused tasks.
2. List `show-options`, `choose-path`, `show-command-map`, `next-step`, and `help`.
3. Present several representative hypothesis, empirical, paper, revision, rebuttal, and readiness examples from [show-options.md](show-options.md).
4. Offer [show-command-map.md](show-command-map.md) only when the complete pass inventory is requested.
5. End with one exact public invocation or the smallest missing decision.

If the request does not map cleanly to these steps, use the native planning tool to build concise help from the typical patterns, pass map, readiness boundary, and user goal.

Use `$isomer-ext-deepsci-welcome` to learn. Use `$isomer-ext-deepsci-entrypoint use <command> to <task>` or a concrete task-only entrypoint request to act. An example is guidance, not readiness evidence or mutation and Gate approval.
