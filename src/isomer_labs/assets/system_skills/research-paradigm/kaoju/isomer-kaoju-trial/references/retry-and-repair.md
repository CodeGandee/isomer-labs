# Retry and Repair

An identical retry is allowed only for a classified transient failure and within the accepted attempt bound. Preserve the same plan, commit, lock, data, wrapper, evaluator, metrics, resources, and interpretation, and record a new Run.

A dependency or lock change, source commit or patch, data change, wrapper or entry-point semantic change, evaluator or metric change, resource-limit change, fidelity change, or evidence-interpretation change is material. Record the failed attempt, revise `kaoju:method-trial-plan`, explain the difference, and obtain another human Gate before executing.
