# Kaoju Welcome Help

## Workflow

1. Explain that Kaoju welcome teaches evidence-led survey patterns and remains read-only, while the Kaoju entrypoint executes managers and bounded procedures.
2. List `show-options`, `choose-path`, `show-command-map`, `next-step`, and `help`.
3. Present representative framing, landscape, reading-list, source-intake, comparison, trial, paper, and wiki examples from [show-options.md](show-options.md).
4. Explain discover versus acquire versus examine, bounded trial versus genuine reproduction, and the role of later Gates.
5. Offer [show-command-map.md](show-command-map.md) only when the complete inventory is requested, then end with one exact public invocation or missing decision.

If the request does not map cleanly to these steps, use the native planning tool to build concise help from the typical patterns, command map, evidence boundaries, interaction Gates, and user goal.

Use `$isomer-ext-kaoju-welcome` to learn. Use `$isomer-ext-kaoju-entrypoint use <command> to <task>` or a concrete task-only entrypoint request to act. An example is guidance, not source selection, execution, network, publication, or mutation approval.
