# Recommend the Next Kaoju Step

## Workflow

1. Identify the current survey stage, accepted source and evidence refs, desired output, and known Project or workspace readiness.
2. Use only read-only Project, extension, survey, dataset, Run, Gate, or durable-record inspection that materially affects the recommendation, and announce it first.
3. If Kaoju Mindset Sources are uninitialized, report that posture and recommend `create-topic` or the intended concrete action; do not enumerate topics, invoke create-missing, create a Run, or write a Source.
4. Match the result to one pattern in [show-options.md](show-options.md).
5. Recommend one public Kaoju command, one task-only Kaoju request, or one public prerequisite-recovery route.
6. Name the next material interaction or Gate, expected mutation, blocker, and exact resume point without executing the route.

If the task does not map cleanly to these steps, use the native planning tool to separate missing scope or source choices from missing readiness evidence, then ask for the smallest decision or recommend the narrowest read-only check.

Do not fabricate evidence or readiness. Discovery, acquisition, examination, trial, comparison, audit, synthesis, and output are distinct stages with durable handoffs. The first recommended invocation does not authorize a later governed action.
