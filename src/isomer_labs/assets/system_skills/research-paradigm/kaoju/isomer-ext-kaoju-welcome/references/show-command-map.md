# Show Kaoju Command Map

## Workflow

1. Load this exhaustive map only for `show-command-map`, `help`, or explicit complete output.
2. Present every current `isomer-ext-kaoju-entrypoint` public command exactly once in manifest order.
3. Keep grouped manager operations expressed through the accepted public command form rather than private object-generator notation.
4. State that the initial command does not preapprove later clarification, source, execution, network, template, publication, or mutation Gates.

If the goal does not map cleanly to one command, use the native planning tool to compare intent, evidence stage, output, and interaction boundary, then recommend a task-only Kaoju entrypoint request or ask for the missing decision.
