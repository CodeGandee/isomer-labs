# Human Gates for Irreversible or Claim-Shaping Decisions

Isomer Labs will return to the human user for irreversible or claim-shaping decisions rather than gating every workflow stage. The default gate set includes goal framing, team and workflow approval, baseline acceptance or waiver, experiment route selection, claim strength, finalization, and destructive or archival actions.

## Status

accepted

## Considered Options

- Ask the user at every stage boundary.
- Gate only irreversible or claim-shaping decisions.
- Let teams run autonomously by default and rely mainly on user interrupts.

## Consequences

- Routine team execution can continue without turning every stage transition into a manual approval step.
- The operator agent needs a policy surface that can classify decisions by reversibility, user preference, evidence risk, and claim impact.
- The GUI must expose pending gates clearly, including the evidence, options, and consequences behind each requested decision.
