# Kaoju Topic Derived-Intent Use Cases

These use cases describe how a Project user creates a Kaoju Research Topic, reviews its editable derived intent, and applies later edits without rewriting past research material.

## Index

| ID | Use Case | Status | Primary User Question |
| --- | --- | --- | --- |
| [UC-01](uc-01-create-review-and-apply-kaoju-derived-intent.md) | Create, Review, and Apply Kaoju Derived Intent | Proposed | What can I adjust after topic creation, and how do my edits affect future work without changing past work? |

## Notes

- UC-01 covers both the review pause immediately after explicit Kaoju topic initialization and a later return after the Topic Workspace has been used.
- Mindset Sources are directly editable current topic intent, while writing-template directories are non-canonical working copies whose edits require promotion into named template Artifact records.
- Generated environment target specifications or other service-owned derived files are not accepted as direct user overrides; the agent redirects adjustments to their source intent and regeneration owner.
- The OpenSpec baseline is [proposal.md](../../proposal.md), with ownership and fallback decisions in [design.md](../../design.md).
