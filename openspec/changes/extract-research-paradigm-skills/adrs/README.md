# ADRs for Extract Research Paradigm Skills

These ADRs capture the concept-mapping decisions for migrating DeepScientist-derived research behavior into Isomer Labs research-paradigm skills.

The decision boundary is:

- Isomer Labs domain language is authoritative.
- DeepScientist concepts are source-system references, not requirements to add Isomer concepts.
- DeepScientist continuation scheduling is a source runtime detail, not an Isomer research-paradigm concept.
- A missing DeepScientist equivalent is a review item until an Isomer ADR or spec accepts it.
- Unsettled concrete paths, files, commands, providers, APIs, schemas, and generated layouts must be marked with `yet-to-be-determined`.
